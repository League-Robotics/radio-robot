# T002: Config params (Config.h, gen_default_config.py, DefaultConfig.cpp)

## Description

Add three EKF noise configuration fields to `RobotConfig` and wire them through
the config generation pipeline. These fields carry the tuned EKF parameters
(from the sprint 021 demo notebook) into the firmware as defaults. T003 and T004
depend on these fields being present in `RobotConfig`.

## Acceptance Criteria

- [x] `source/types/Config.h` has three new float fields in `RobotConfig` added
  after the `otosGate` line:
  ```cpp
  // EKF sensor fusion parameters
  float ekfQxy;       // process noise: position (mm^2) — default 2.0
  float ekfQtheta;    // process noise: heading (rad^2) — default 0.005
  float ekfROtosXy;   // OTOS measurement noise: position (mm^2) — default 50.0
  ```
- [x] `scripts/gen_default_config.py` has the three EKF fields added to the
  `generate()` function's f-string template, near the `alphaPos` block:
  ```cpp
  // EKF sensor fusion
  p.ekfQxy         = 2.0f;
  p.ekfQtheta      = 0.005f;
  p.ekfROtosXy     = 50.0f;
  ```
- [x] `source/robot/DefaultConfig.cpp` has been regenerated by running
  `python3 scripts/gen_default_config.py` and contains the three new assignments.
- [x] `defaultRobotConfig()` compiles cleanly and returns a config with
  `ekfQxy = 2.0f`, `ekfQtheta = 0.005f`, `ekfROtosXy = 50.0f`.
- [x] Full firmware build (`python3 build.py`) succeeds with no errors.
- [x] Full test suite passes: `uv run --with pytest python -m pytest`
