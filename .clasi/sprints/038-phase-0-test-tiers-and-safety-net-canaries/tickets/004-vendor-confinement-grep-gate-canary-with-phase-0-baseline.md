# Vendor-confinement grep-gate canary with Phase 0 baseline

## Description

Add `tests/simulation/unit/test_vendor_confinement.py` — a pytest test that greps
`source/` above `source/hal/` for forbidden vendor tokens and asserts the hit-set has not
grown vs a committed baseline. Also create and commit the baseline file
`tests/_infra/vendor_baseline.txt` that records the Phase 0 known leaks.

This is the ratchet gate for the migration: Phase A reduces the baseline (sealing
`MotorController`/`DebugCommandable` leaks), Phase B reduces it further, etc. The gate
fails if any NEW violation appears outside the baseline — it never fails on existing
known leaks.

## Forbidden tokens (Phase 0 scope: above `source/hal/`)

Directories to inspect: `source/app/`, `source/control/`, `source/robot/`, `source/types/`.
Do NOT inspect `source/hal/` itself — vendor types belong there.

Forbidden patterns (grep with `-rn`):
1. `#include.*MicroBit\.h` — CODAL core include
2. `I2CBus` — type reference (class name, pointer, field)
3. `microbit_random` — CODAL RNG call
4. Nezha split-phase I2C pattern: `0x46` register references specifically in non-hal context
   (Note: `0x46` is the Nezha motor command register — grep for it combined with the
   context of motor split-phase. Be surgical to avoid false positives.)
5. OTOS raw int16 register access: grep for the SparkFun OTOS raw register suffix pattern
   (e.g., `Raw` in the context of OTOS register reads — the `IOtosSensor` interface methods
   that expose `int16` raw register values like `readRawLinear`, `readRawAngular`, or any
   `*Raw` accessor that returns the LSB-scale integer.)

**Implementation note on token granularity**: Start with the simplest grep that catches
the documented leaks: `MicroBit.h`, `I2CBus`, `microbit_random`. For OTOS `Raw` and Nezha
`0x46`, the programmer should run the grep against the actual source tree first, inspect
the hits, and tune the pattern to avoid false positives before committing the baseline.
The goal is zero false positives at Phase 0 green.

## Baseline file format (`tests/_infra/vendor_baseline.txt`)

One hit per line, format: `relative/path/to/file.cpp:lineno: matched text (truncated)`.
Example:
```
source/control/MotorController.h:3:#include "MicroBit.h"
source/control/MotorController.cpp:2:#include "MicroBit.h"
source/app/DebugCommandable.cpp:27:  I2CBus* _bus;
```

The baseline is generated by running the grep at Phase 0, committing the output, and
then the test asserts that no NEW lines appear relative to the baseline. Existing baseline
lines may disappear (Phase A sealing them) without failing the gate.

## Test structure (`test_vendor_confinement.py`)

```python
# Pseudocode — programmer implements:
import pathlib, re, pytest

REPO_ROOT = pathlib.Path(__file__).parents[3]   # tests/simulation/unit/ → repo root
SOURCE_DIR = REPO_ROOT / "source"
INSPECT_DIRS = ["app", "control", "robot", "types"]  # above hal/
BASELINE_FILE = REPO_ROOT / "tests" / "_infra" / "vendor_baseline.txt"

FORBIDDEN_PATTERNS = [
    r'#include.*MicroBit\.h',
    r'\bI2CBus\b',
    r'\bmicrobit_random\b',
    # Add OTOS Raw and Nezha 0x46 patterns after empirical tuning
]

def collect_hits():
    hits = set()
    for d in INSPECT_DIRS:
        for f in (SOURCE_DIR / d).rglob("*.[ch]pp"):
            for lineno, line in enumerate(f.read_text(errors="replace").splitlines(), 1):
                for pat in FORBIDDEN_PATTERNS:
                    if re.search(pat, line):
                        rel = f.relative_to(REPO_ROOT)
                        hits.add(f"{rel}:{lineno}: {line.strip()[:80]}")
    return hits

def test_vendor_confinement_no_new_leaks():
    baseline = set(BASELINE_FILE.read_text().splitlines()) if BASELINE_FILE.exists() else set()
    current = collect_hits()
    new_leaks = current - baseline
    assert not new_leaks, (
        f"New vendor leaks found above source/hal/ ({len(new_leaks)} new):\n"
        + "\n".join(sorted(new_leaks))
        + "\nUpdate tests/_infra/vendor_baseline.txt ONLY if these are intentional."
    )
```

## Baseline generation procedure

1. Run `python3 -c "..."` (or a small script) to collect all current hits.
2. Save the output to `tests/_infra/vendor_baseline.txt`.
3. Run the test — it must pass green with the newly committed baseline.
4. Commit both `test_vendor_confinement.py` and `vendor_baseline.txt` together.
