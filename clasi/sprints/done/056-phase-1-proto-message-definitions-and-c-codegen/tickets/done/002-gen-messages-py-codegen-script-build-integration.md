# gen_messages.py codegen script + build integration

## Description

Write `scripts/gen_messages.py` ظ¤ the host-side code generator that parses the
proto3 files from ticket 001 and emits C++11 POD headers to `source/messages/*.h`.
Wire it into `build.py` alongside the existing `gen_default_config.py` call.
Add `grpcio-tools` as a dev dependency in `pyproject.toml`.

The generator pattern mirrors `scripts/gen_default_config.py`: run once before
the compiler, emit deterministic output, fail fast if inputs are invalid.

## Generator behaviour specification

### Input

`protos/*.proto` files (all 7 from ticket 001 plus `protos/options.proto`).

### Processing

Use `grpcio-tools` Python API (not subprocess `protoc`) to obtain a
`FileDescriptorSet`:
```python
from grpc_tools import protoc
protoc.main([
    '-I', 'protos',
    '-I', '<grpcio-tools well_known_protos path>',
    '--descriptor_set_out=<tmp file>',
    '--include_imports',
    'protos/common.proto', ...
])
```
Then parse the `FileDescriptorSet` with `google.protobuf.descriptor_pb2`.

(field) and emit C++ as follows:

### Output rules (emit to `source/messages/<name>.h`)

**Preamble** (in `common.h` only, included by all others):
```cpp
// AUTO-GENERATED ظ¤ do not edit by hand.
// Regenerated by scripts/gen_messages.py before each firmware build.
#pragma once
#include <stdint.h>

template<class T>
struct Opt { bool has = false; T val{}; };
```

All other generated headers `#include "messages/common.h"`.

**Scalar fields** (int32/uint32/bool/float/string-as-char-array):
```cpp
float x_mm = 0.0f;
uint32_t count = 0;
bool connected = false;
```

**Nested message fields**:
```cpp
Pose2D pose = {};
```

**`oneof` fields** ظْ Kind enum + union-like struct:
```cpp
enum class ControlKind : uint8_t { NONE = 0, TWIST = 1, WHEELS = 2, NEUTRAL = 3, POSE = 4 };
ControlKind control_kind = ControlKind::NONE;
union {
    BodyTwist3 twist;
    WheelTargets wheels;
    Neutral neutral;
    SetPose pose;
} control = {};
```
No RTTI. Caller inspects `control_kind` before reading union member.

**Optional (nullable) fields** ظْ `Opt<T>`:
```cpp
Opt<float> feedforward = {};
Opt<float> position_mm = {};
```

**`repeated` with `(max_count)=N`** ظْ fixed array + count:
```cpp
WheelTarget w[4] = {};
uint8_t w_count = 0;
```
The `N` value is read from the `max_count` field option in the descriptor.

**Getters** (on all message types):
```cpp
const float& x_mm() const { return x_mm_; }
const Opt<float>& feedforward() const { return feedforward_; }
// etc.
```
Use private members with trailing `_` suffix and public getter/setter accessors
(or plain public members with a comment ظ¤ implementer's choice, as long as the
round-trip test in ticket 003 passes).

**Chainable setters** (Command and Config message types ONLY):
```cpp
DrivetrainCommand& setTwist(float vx, float vy, float omega) {
    control_kind = ControlKind::TWIST;
    control.twist = {vx, vy, omega};
    return *this;
}
DrivetrainConfig& setVelKp(float v) { vel_gains_.kp = v; return *this; }
```

**No** `std::`, `new`, `delete`, `throw`, `dynamic_cast`, `typeid` in any generated file.

### `--emit-inventory` flag

When `--emit-inventory` is passed, also write `docs/design/message-inventory.md`
with a Markdown table:

| Proto file | Message | Field | C++ type | Maps to existing |
|---|---|---|---|---|
| drivetrain.proto | DrivetrainState | fused | PoseEstimate | ActualState::fused |
| ... | | | | |

The "Maps to existing" column is populated from a hand-authored mapping table
inside `gen_messages.py` (a Python dict: `{('DrivetrainState','fused'): 'ActualState::fused', ...}`).
This mapping dict is the main editorial work of ticket 004.

### `--dry-run` flag

Print what would be written without writing files. Useful for CI verification.

## Implementation Plan

### Approach

Model `gen_messages.py` closely on `gen_default_config.py`:
- `REPO_ROOT = Path(__file__).resolve().parent.parent`
- `PROTO_DIR = REPO_ROOT / "protos"`
- `OUT_DIR = REPO_ROOT / "source" / "messages"`
- Emit to `OUT_DIR / f"{name}.h"` for each `.proto` file.

Use a descriptor walk rather than text parsing to remain stable across proto syntax variants.

### Files to create

- `scripts/gen_messages.py`
- `source/messages/` directory (the generator creates it if absent)

### Files to modify

- `build.py` ظ¤ add second `_sp.run([sys.executable, _gen_msgs], check=True)` call
  after the existing `gen_default_config.py` call (around line 89 in the current file).
- `pyproject.toml` ظ¤ add `grpcio-tools>=1.60,<2.0` to dev dependencies.

### Testing plan

Run `python scripts/gen_messages.py` manually and inspect the output headers.
Verify determinism: `python scripts/gen_messages.py && python scripts/gen_messages.py`
produces no diff.
Run `uv run python -m pytest` to confirm no regressions (no new tests added in this ticket).

Compile verification is ticket 003's responsibility.

### Documentation updates

None beyond the script docstring. The traceability table is ticket 004.
