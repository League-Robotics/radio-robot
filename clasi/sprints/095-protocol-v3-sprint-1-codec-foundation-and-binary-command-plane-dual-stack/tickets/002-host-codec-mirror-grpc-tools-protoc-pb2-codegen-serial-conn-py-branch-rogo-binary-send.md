# Host codec mirror: grpc_tools.protoc pb2 codegen + serial_conn.py * branch + rogo binary send

## Description

Give the host the identical schema ticket 001 declared (compiled Python
bindings via `grpc_tools.protoc`, the SAME tool `gen_messages.py` already
uses on the firmware side) and a receive/send path for the binary plane
over the exact transport/demux the text plane already uses. This ticket
is sequenced right after 001 (not at the end) because ticket 006's
differential test harness needs `host/robot_radio/robot/pb2/` as its
`google.protobuf` reference implementation.

1. **`grpc_tools.protoc --python_out=host/robot_radio/robot/pb2`** wired
   into `justfile`/`build.py` beside the existing `gen_messages.py`
   invocation — same `protos/*.proto` source, same `grpc_tools`
   dependency already in `pyproject.toml`'s codegen group (verified:
   `grpcio-tools>=1.60,<2.0` already present). Device tables and host pb2
   bindings can never skew independently since both compile from the
   identical proto source on every build.
2. **`host/robot_radio/io/serial_conn.py`'s `_reader_loop`** gains ONE new
   branch, inserted before the final "anything else: drop silently"
   fallthrough: a line starting with `*` is dearmored (strip `*B` prefix,
   base64-decode), parsed via `pb2.ReplyEnvelope.FromString(...)`, and
   routed through the SAME `_reply_queues`/`_tlm_queue` machinery the
   existing `OK`/`ERR`/`CFG`/`ID`/`TLM` branches already use (keyed by
   `corr_id`, same `_CORR_ID_RE`-equivalent lookup — the envelope's
   `corr_id` field replaces the text plane's `#<digits>` suffix
   extraction, same semantics). **Every existing branch
   (`TLM`/`EVT`/`OK`/`ERR`/`CFG`/`ID`/`#`-comment/`keepalive`) is
   untouched** — verify by diff, not by inspection alone.
3. **`SerialConnection.send_envelope(envelope: pb2.CommandEnvelope,
   read_timeout=500, ...)`** alongside the existing text `send()` —
   serializes, base64-armors (`*B<base64>\n`), writes, and blocks on the
   corr-id-keyed reply queue exactly like `send()` does today (reuse
   `send()`'s retry-on-corrupted-command logic if it transfers cleanly;
   note in completion notes if it doesn't and a simpler non-retrying
   version was shipped instead).
4. **`host/robot_radio/io/cli.py` (`rogo`)** gains a binary send path
   (e.g. a `--binary`/`-b` flag or a dedicated subcommand) that can build
   and send a `CommandEnvelope` for each of this sprint's seven
   implemented arms (drive/segment/replace/stop/ping/echo/id).

## Acceptance Criteria

- [ ] `host/robot_radio/robot/pb2/` is generated by `grpc_tools.protoc`
      from `protos/*.proto` (via `justfile`/`build.py`) and is importable
      (`from robot_radio.robot.pb2 import envelope_pb2`, or whatever
      module layout `protoc`'s `--python_out` naturally produces — do not
      hand-restructure its output).
- [ ] `SerialConnection._reader_loop`'s new `*`-branch correctly
      classifies a `*`-prefixed line, dearmors + parses it as a
      `ReplyEnvelope`, and delivers it to the corr-id-keyed caller's
      queue exactly as an `OK ... #<id>` text reply is delivered today
      (verify with a unit test that feeds a synthetic `*`-line into the
      reader's classify logic directly, without needing live hardware).
- [ ] `send_envelope()` round-trips against a synthetic loopback (no live
      hardware needed) or against ticket 006's differential harness's
      host-side codec.
- [ ] `rogo` gains a working binary send path for each of
      drive/segment/replace/stop/ping/echo/id.
- [ ] **No existing `_reader_loop` branch changes behavior** — diff
      `serial_conn.py` before/after and confirm the `TLM`/`EVT`/`OK`/
      `ERR`/`CFG`/`ID`/`#`-comment/`keepalive` branches are byte-identical
      except for the one new `*`-branch insertion point.
- [ ] `just build-sim` and the full existing sim suite stay green (this
      ticket touches host/ only, but verify anyway — no firmware/tests
      regression is expected or acceptable).

## Testing

- **Existing tests to run**: any existing `host/`/`robot_radio` test
  suite covering `SerialConnection` (`uv run python -m pytest` — locate
  via `grep -rl SerialConnection tests/`), plus `tests/sim -q` as a
  blanket regression check.
- **New tests to write**: a `_reader_loop` classification unit test
  feeding a synthetic `*B<base64>\n` line and asserting it lands in the
  correct `_reply_queues[corr_id]` entry; a `send_envelope()` round-trip
  test against a loopback/mock serial object; an import smoke test for
  `host/robot_radio/robot/pb2/`.
- **Verification command**: `uv run python -m pytest tests/sim -q` plus
  the new host-side test file(s) this ticket adds (e.g. `uv run python -m
  pytest host/ -k serial_conn` or wherever the new tests land per this
  project's existing host test layout).
